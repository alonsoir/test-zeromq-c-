=== DEBT-FD-001: Fast Detector hardcoded thresholds ===
FastDetector::is_suspicious() (Path A, DAY 13) uses compiled constants,
ignoring sniffer.json entirely. Violates "JSON is the law".

  fast_detector.hpp:
    THRESHOLD_EXTERNAL_IPS = 10  (JSON has external_ips_30s: 15 — Path B only)
    THRESHOLD_SMB_CONNS    = 3   (JSON has smb_diversity: 10 — Path B only)
    THRESHOLD_PORT_SCAN    = 10  (no JSON equivalent)
    THRESHOLD_RST_RATIO    = 0.2 (no JSON equivalent)
    WINDOW_NS              = 10s (no JSON equivalent)

Dual-path architecture:
  Path A: per-packet, hardcoded constants, 10s window → [FAST ALERT] log
  Path B: aggregated, JSON config, 30s window → send_ransomware_features()

Fix target: PHASE2. Migrate to FastDetector constructor injection.
Documented in docs/adr/ADR-006-fast-detector-hardcoded-thresholds.md

=== ML DETECTOR: Three counters, three semantics ===
  log "🚨 ATTACK"       → label_l1 == 1 (binary RF vote, no threshold)
  stats_.attacks_detected → label_l1==1 AND conf >= level1_attack(0.65)
  final_classification   → final_score >= malicious_threshold (JSON)

  level1_attack=0.65 lives in ml-detector/config/ml_detector_config.json
  (not in sniffer.json). Documented, not a bug.
