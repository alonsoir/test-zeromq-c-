feat(day18): implement bidirectional put_config() with encryption support

## Summary
Implements complete bidirectional configuration management between etcd-client
and etcd-server, enabling components to programmatically upload their configs
with automatic compression and encryption.

## Changes

### etcd-client library
- Add http::put() function with retry logic (http_client.cpp)
- Implement EtcdClient::put_config() method (etcd_client.cpp)
- Integrate with process_outgoing_data() for auto compression/encryption
- Forward declarations updated for http::put()

### etcd-server
- Add PUT /v1/config/:id endpoint (etcd_server.cpp)
- Support dual content types:
  * application/octet-stream (encrypted+compressed)
  * application/json (plain JSON for Phase 1 MVP)
- Auto-decrypt with component_registry_->decrypt_data()
- Store via register_component()

## Architecture
```
Client: JSON → Compress(LZ4) → Encrypt(ChaCha20) → HTTP PUT
Server: HTTP → Decrypt → Decompress → Validate → Store
```
